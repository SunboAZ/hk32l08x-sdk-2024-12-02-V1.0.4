******************************例程详细描述*****************************
功能描述:
					1. 配置HSE+PLL 48M做为系统时钟。
					2. 通过MCO引脚（PA8）输出系统时钟。
					3. 使用CSS(Clock Security System)特性检测HSE故障，当HSE时钟消失(外部石英损坏或断开)，
					   切换HSI作为系统时钟源，并产生中断(NMI)。在NMI中断中，再次使能HSE，并且使能HSERDY
					   中断，一旦HSE时钟恢复，就会产生HSERDY中断，在该中断中，系统时钟被重新配置到之前的状态
					4. 两个LED延时闪烁。
					5. 通过RCC_ClockFreq结构体可观察当前不同外设运行的时钟。
					
时钟配置:
	本例程采用HSE+PLL48MHz作为系统时钟，用户如果需要更改系统时钟配置，请参考对应用户手册或者
	RCC库函数及RCC相关例程。

适用芯片:
	HK32L0xx全系列

涉及外设:
	RCC、GPIO
	
涉及管脚:	
	MCO -----> PA8 
	LED1 ----> PF4 
	LED2 ----> PF5
	
注意事项:
	1. 推荐使用MDK5.00及其以上版本编译本工程（MDK-ARM）
	          或者IAR8.22及其以上版本编译本工程(EWARM);
	2. 使用之前检测电源VCC、GND等是否正常。
如何使用:
	1. 直接编译程序下载到评估版，重启评估版;
	2. 观察两颗LED是否闪烁，PA8是否正确输出系统时钟。当断开外部晶振时，
	   通过PA8观察系统时钟是否被正确切换到HSI上，两颗LED是否闪烁；当再次
	   连接外部晶振时，通过PA8观察系统时钟是否被正确切换回原来状态，两颗LED是否闪烁。
