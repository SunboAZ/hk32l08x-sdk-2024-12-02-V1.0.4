******************************例程详细描述*****************************
功能描述:
	本示例描述了如何配置CAN的两个FIFO进行接收和发送CAN帧，本例程在三个Board之间进行CAN的收发。
	将CAN FIFO0过滤器配置为只接收帧ID为0x321的CAN帧，FIFO1过滤器配置为只接收帧ID为0x123的CAN帧。
	当第一次按下KEY1，在中断中发送一帧标准帧ID:0x321 DLC:1 DATA:0x04，此后连续按下KEY1时，每按下一次KEY1则DATA减一，若DATA小于0x01，则DATA重新赋值为0x04；
	当第一次按下KEY2，在中断中发送一帧标准帧ID:0x123 DLC:1 DATA:0x01，此后连续按下KEY2时，每按下一次KEY2则DATA加一，若DATA大于0x04，则DATA重新赋值为0x01；
	当A或B、C板按下按键KEY1或KEY2时,会有对应的LED亮灭，同时A板或B板、C板接收到CAN帧后，有相同的LED亮灭，具体如下：
	DATA:0x01---------->LED1亮
	DATA:0x02---------->LED2亮
	DATA:0x03---------->LED1循环亮灭
	DATA:0x04---------->LED2循环亮灭

时钟配置:
	本例程采用HSE+PLL48MHz作为系统时钟，用户如果需要更改系统时钟配置，请参考对应用户手册或者
	RCC库函数及RCC相关例程。

适用芯片:
	HK32L0xx全系列

涉及外设:
	CAN/GPIO
	
涉及管脚:	
	KEY1 -----------> PB2
	KEY2 -----------> PC13
	LED1 -----------> PF4
	LED2 -----------> PF5
	CAN RX -----------> PA11
	CAN TX -----------> PA12

注意事项:
	1. 推荐使用MDK5.00及其以上版本编译本工程（MDK-ARM）
	          或者IAR8.22及其以上版本编译本工程(EWARM);
	2. 使用之前检测电源VCC、GND等是否正常。
如何使用:
	1. 直接编译程序下载到评估版，重启评估版;
	2. 按下A或B、C板按键KEY1或KEY2观察对应的B或A、C板LED亮灭情况
