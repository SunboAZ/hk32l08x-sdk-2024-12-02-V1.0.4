******************************例程详细描述*****************************
功能描述:
	本例程描述了CAN总线上的多个节点之间进行通信。
	通过调用库函数CAN_Init()和CAN_FilterInit()来完成对CAN的配置，ID配置为0x321，DLC配置为1，每次按下按键KEY1使得定义的变量KeyNumber自加1，
	同时将KeyNumber作为data[0]发送出去，CAN总线上的其它单元收到后同时会点亮相同的LED。
	示例：对三个评估板A、B、C下载相同的程序，下载完成后重新复位，按下板A的按键KEY1，则KeyNumber为1，
	发送一帧标准帧 ID:0x321 DLC:1 DATA: 0x01，B和C接收到后，在CAN中断中判断ID和DLC接收正确后，使KeyNumber=DATA[0]=0x01，则LED1亮，
	由于只有2个LED灯，当KeyNumber为0x02时，LED2亮；当KeyNumber为0x03时，LED1循环亮灭；当KeyNumber为0x04时，LED2循环亮灭

时钟配置:
	本例程采用HSE+PLL48MHz作为系统时钟，用户如果需要更改系统时钟配置，请参考对应用户手册或者
	RCC库函数及RCC相关例程。

适用芯片:
	HK32L0xx全系列

涉及外设:
	CAN/GPIO/USART
	
涉及管脚:	
	KEY1 -----------> PB2
	LED1 -----------> PF4
	LED2 -----------> PF5
	CAN RX -----------> PA11
	CAN TX -----------> PA12

注意事项:
	1. 推荐使用MDK5.00及其以上版本编译本工程（MDK-ARM）
	          或者IAR8.22及其以上版本编译本工程(EWARM);
	2. 使用之前检测电源VCC、GND等是否正常。
如何使用:
	1. 直接编译程序下载到评估版，重启评估版;
	2. 将3个评估板以总线方式连接，当其中一个评估版按下按键KEY1发送标准帧后，观察另外两个评估板的相同的LED是否亮
