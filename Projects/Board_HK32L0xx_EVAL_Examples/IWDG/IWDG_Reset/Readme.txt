******************************例程详细描述*****************************
功能描述:
					1. 这个例子展示了如何配置IWDG，定时喂狗，以及如何模拟软件故障停止喂狗，在故障时IWDG复位。
					2. 当在main.h中启用宏“LSI_TIM_MEASURE”时，可以通过TIM14来获取到准确的LSI时钟，从而配置准确的
					   IWDG超时时间。
					3. 当宏“LSI_TIM_MEASURE”定义被禁用时，LSI频率被设置为32.768Khz，会产生较不准确的IWDG超时
					4. 配置IWDG超时时间为250ms，在主循环中，间隔220ms喂一次狗，LED2也每220ms翻转一次，表明程序正在运行。
					5. 在KEY1按下时，代码对无效地址进行写入操作，会产生一个Hardfault异常，导致IWDG停止喂狗复位，复位后
					   LED1亮起。

时钟配置:
	本例程采用HSE+PLL48MHz作为系统时钟，用户如果需要更改系统时钟配置，请参考对应用户手册或者
	RCC库函数及RCC相关例程。

适用芯片:
	HK32L0xx全系列

涉及外设:
	IWDG、GPIO、RCC、TIM
	
涉及管脚:	
	LED1 ----> PF4 
	LED2 ----> PF5
	KEY1 ----> PB2

注意事项:
	1. 推荐使用MDK5.00及其以上版本编译本工程（MDK-ARM）
	          或者IAR8.22及其以上版本编译本工程(EWARM);
	2. 使用之前检测电源VCC、GND等是否正常。
如何使用:
	1. 直接编译程序下载到评估版，重启评估版;
	2. 观察LED2是否正常闪烁。当KEY按下后，观察LED1是否亮起，LED2是否继续闪烁。
